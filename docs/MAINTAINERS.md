# 🛡️ Maintainers Guide

Guía breve para mantenedores del proyecto.

## Rol del maintainer

Un maintainer en `docker-labs` debe cuidar:

- coherencia entre código y documentación
- dirección de la plataforma principal
- calidad mínima de los labs
- claridad para usuarios y contribuyentes

## Responsabilidades

- revisar PRs
- decidir si un nuevo lab realmente aporta
- proteger la narrativa `platform / infra / starter`
- evitar que la documentación prometa más de lo que el código entrega

## Checklist antes de aceptar cambios

- el entorno levanta de verdad
- la documentación coincide con lo entregado
- el lab o mejora tiene justificación
- no introduce puertos o conflictos innecesarios
- encaja con la dirección del proyecto

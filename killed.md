# Killed Practices

> **Version**: 1.4  
> **Estado**: Activo  
> **Uso recomendado**: Documento de criterio para entender que enfoques el proyecto evita de forma intencional

---

## Practicas descartadas

| Practica | Por que se evita |
|---|---|
| Tratar cada carpeta como demo aislada sin narrativa comun | Le resta valor de producto al repo |
| Forzar el mismo CRUD en todos los labs | Homogeneiza en exceso y borra el valor de cada stack |
| Vender el proyecto como reemplazo completo de Docker Desktop | Hoy el repo no busca eso |
| Documentacion inflada sin verificacion real | Deteriora credibilidad tecnica |
| Levantar todo por defecto en equipos justos | Castiga recursos y empeora la experiencia |

## Criterio actual

El repositorio busca ser una plataforma modular, profesional y util para aprender, operar y mostrar criterio tecnico.

# Security Policy

## Reportar una vulnerabilidad

Si encuentras una vulnerabilidad de seguridad en este proyecto, por favor repórtala de forma confidencial a: [TU CORREO] (reemplaza con un correo real). Envía:

- Una descripción clara del problema
- Pasos para reproducirlo
- Impacto estimado
- Posible parche o mitigación

Si prefieres, puedes abrir un *issue* marcado como `security` pero evita publicar detalles sensibles públicamente.

## Respuesta y tiempos

- Confirmación de recepción: 48 horas
- Evaluación y plan de mitigación: 7 días laborables (dependiendo de la gravedad)

## Soportadas versiones

Mantén en este documento una tabla o lista con las versiones soportadas del software incluido (por ejemplo: versión de PHP/Node/Python presentes en los labs).

## Divulgación responsable

Se solicita responsable divulgación: se dará un plazo razonable para parchear antes de la publicación de detalles técnicos.

## Claves PGP

Si deseas enviar información cifrada, añade la clave PGP aquí o en el contacto. (Opcional)

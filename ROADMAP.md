# ROADMAP

## Visión

Hacer de `docker-labs` un repositorio didáctico y reproducible con laboratorios prácticos que cubran:

- Docker + Docker Compose (fundamentos)
- Stacks comunes: Node, Python, LAMP
- Integración con bases de datos y cola (Redis/RabbitMQ)
- Observabilidad básica (logs y métricas)
- Buenas prácticas en CI/CD y seguridad mínima

## Prioridades (corto plazo)

- [ ] Documentación y estandarización de plantillas (Dockerfile, .env.example)
- [ ] Añadir tests básicos para los ejemplos de API
- [ ] Integrar linters y formateadores para cada stack

## Objetivos (mediano plazo)

- [ ] Añadir laboratorios con autenticación y persistencia
- [ ] Casos de ejemplo de despliegue con Traefik/Nginx
- [ ] Integración con GitHub Actions para pruebas y build

## Ideas a largo plazo

- [ ] Ejemplos de monitorización con Prometheus/Grafana
- [ ] Laboratorios sobre multi-host y orquestadores (Kubernetes)

## Cómo contribuir al roadmap

1. Abre un *issue* proponiendo una nueva funcionalidad o mejora.
2. Marca la prioridad y el impacto esperado.
3. Mantendremos esta hoja de ruta sincronizada con los *issues* etiquetados `roadmap`.
